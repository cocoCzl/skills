# 足球竞彩助手

`football-betting-assistant` 是一个面向中文足球竞彩分析的 agent-first skill 包。它用于胜平负、让球胜平负、比分、大小球、总进球、串关和赛后复盘等场景。

## 使用方式

把 `football-betting-assistant/` 目录放到 Codex 或其他 agent 的 skills 目录中。用户提出足球投注分析、竞彩、胜平负、让球胜平负、比分推荐、大小球、总进球、串关、四串一或赛后复盘相关请求时，这个 skill 应自动触发。

当 agent 可以使用浏览器、搜索、本地文件、已配置的足球数据 API 或内置计算脚本时，效果最好。无法获取实时数据时，它应该只向用户索取最少必要数据，而不是编造赛程、赔率、首发、伤停或天气。

## 能做什么

- 根据自然语言请求发现和确认具体比赛。
- 在工具可用时核验当前赔率、盘口、球队近况、伤停、首发、天气和赛事背景。
- 使用透明模型：预期进球、贝叶斯修正、泊松比分概率和赔率隐含概率对比。
- 输出单场分析、组合分析、比分覆盖、参考购买方案和赛后复盘。

## 不做什么

- 不保证赛果。
- 不提供投注金额、资金分配或 Kelly 仓位建议。
- 不提供追损、回本或翻本建议。
- 不登录投注平台账号，不绕过访问限制。
- 不硬编码足球数据凭据或大模型 API key。

## 示例提问

```text
帮我分析今晚西班牙 vs 沙特，给胜平负、让球、大小球和三个比分候选。
```

```text
明天早上四场世界杯比赛，帮我做一个四串一参考购买方案，重点看比分和大小球。
```

```text
现在不能联网。我给你赔率和球队近况，你按离线模式分析法国 vs 日本有没有价值。
```

```text
A站说德国让1球，B站说德国让1.25；A站说主力前锋伤缺，B站没提。帮我分析风险。
```

```text
复盘一下我昨天买的四串一为什么没中，重点看模型偏差和数据偏差。
```

## 输入模板

当 agent 无法核验当前数据时，可以按下面模板补充信息：

```markdown
比赛：
开赛时间：
赛事：
主客/中立：
想看玩法：胜平负 / 让球胜平负 / 比分 / 大小球 / 四串一
赔率/盘口：
伤停/首发：
近期状态或数据：
```

## 输出内容

单场报告应包含：

- 数据确认：赛程、赔率/盘口、近期状态、xG 或进失球数据、伤停、首发、天气、战意和数据缺口。
- 结论摘要：胜平负、让球、大小球、参考等级、模型置信度、数据置信度。
- 概率判断：胜平负概率和大小球概率。
- 模型依据：预期进球、泊松比分集中、贝叶斯修正，以及有赔率时的赔率价值对比。
- 三个比分候选：主推、次选和冷门/风险比分。
- 参考购买方案：非保证性的参考方案，以及不建议纳入的项目。
- 风险点：临场首发、单源赔率、数据冲突、市场一致性或其他不确定性。

组合报告应包含比赛确认、每场简版结论、组合相关性检查、主组合/保守替代/激进替代，以及 Pass 或排除项说明。

赛后复盘应对比赛前假设和实际结果，识别模型偏差和数据偏差，并避免任何追损建议。
