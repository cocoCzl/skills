# Football Betting Assistant Prompt Examples

这些示例用于安装 skill 后直接向 agent 提问。普通用户不需要手动运行脚本；agent 会按 skill 流程尝试核验赛程、竞彩赔率/盘口、球队上下文，并在数据不足时降级或追问。

## 竞彩四串一

```text
帮我分析北京时间明天早上四场世界杯比赛，给我一个四串一参考购买方案，重点看比分和大小球。
```

预期行为：

- 先把“明天早上”换成明确的北京时间日期和时间段。
- 确认完整比赛清单，不擅自少分析。
- 在 `china-lottery` 模式下优先核验竞彩可买玩法和赔率/盘口。
- 尝试补近期状态、赛程密度、伤停/首发、战意、天气/场地、赔率变化。
- 如果只有 Sporttery 竞彩快照，输出降级的“竞彩快照报告”，不能伪装成完整球队上下文预测。
- 比分票最多选 4 场；大小球/总进球方向最多 8 场；不强行凑满。

## 只看方向，不要比分

```text
帮我看今晚三场竞彩，只看胜平负和让球胜平负，不要比分，给一个保守串关参考。
```

预期行为：

- 不输出比分购买方案。
- 只使用确认可买的胜平负或让球胜平负市场。
- 如果普通胜平负不可买，只能作为概率分析，不能写入购买方案。

## 比分和总进球

```text
帮我分析周三080、周三081、周三082，重点看比分覆盖和总进球，给基础覆盖和增强覆盖。
```

预期行为：

- 按竞彩编号匹配比赛。
- 比分以候选集合呈现，不给单点确定答案。
- 若比分或总进球赔率缺失，必须标明缺口；可以做概率倾向，但不能做完整赔率价值判断。

## 半全场显式请求

```text
帮我看周三080的竞彩半全场，重点看胜胜、平胜、平平有没有价值。
```

预期行为：

- 半全场只在这种明确请求下启用。
- 需要 verified HAFU 九宫格赔率才能做价值判断或购买方案。
- 没有 HAFU 赔率时只给概率倾向和数据缺口。

## 用户提供截图或赔率

```text
我有三场竞彩截图，里面有让球、比分、总进球赔率。请按截图里的可买玩法分析，不要用第三方赔率替代竞彩赔率。
```

预期行为：

- 优先使用用户截图/文字中的中国竞彩可买玩法。
- 第三方赔率最多作为国际市场参考，不能替代竞彩购买方案。
- 如果截图缺某个玩法，不能把该玩法写入购买单。

## 赛后复盘

```text
帮我复盘最近 30 天生成过的足球竞彩预测报告，看看比分、胜平负、大小球分别命中得怎么样。
```

预期行为：

- 扫描 `data/football/predictions/` 下的 prediction snapshots。
- 核验完赛比分，生成结构化 review JSON 和 HTML 复盘报告。
- 只做校准和流程改进，不输出追损或翻本建议。

## 国际赔率模式

```text
用国际赔率分析今晚西班牙 vs 德国，重点看 h2h、spreads 和 totals 的市场价值，不按中国竞彩口径出单。
```

预期行为：

- 使用 `international-odds` 模式。
- 可用 `THE_ODDS_API_KEY` 或用户授权的国际赔率源。
- 不把国际赔率混作中国竞彩赔率。
